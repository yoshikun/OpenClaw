## 用法

```bash
# 从 workspace 目录执行
node scripts/create_story.mjs "标题" <template> [handler] [description]
```

```javascript
const { createStory } = await import('./scripts/create_story.mjs');
await createStory({
  title: '需求标题',
  template: 'client',   // 默认 client
  handler: '叶枝君',     // 母单处理人，默认 ''
  description: '描述',   // 可选
});
```

## 给已有需求加单一子需求

不嵌套模板，不加预设子工作项，只加一个单一子需求。

```bash
node scripts/add_sub_story.mjs <parent_id> <子需求类型> <标题> [处理人]
# 示例: node scripts/add_sub_story.mjs 1037715 服务器 "新增登录接口" "胡新雷1"
```

可用子需求类型：客户端 / 服务器 / 策划 / QA / UI / UI验收 / 动作 / 特效 / 音频 / 原画

## 核心实现

1. 打开创建页，读取默认模板的 FormData
2. 从 API 获取 1v1 模板的 13 项预设数据（处理人已正确）
3. QA 处理人追加周以天
4. 设置 `data[Template][id] = 1131253609001000047`
5. **清空** FormData 中所有 PresetItems（来自默认模板的 17 项）
6. 从 1v1 模板数据中**只重建**激活索引的 PresetItems（3-4 项）
7. 设置 `data[Story][owner]` 为发单人
8. 用 `fetch` POST 提交
